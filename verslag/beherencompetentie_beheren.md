Competentie Beheren
Inleiding

Tijdens dit transcriptomicsproject heb ik laten zien dat ik onderzoeksgegevens op een gestructureerde, veilige en reproduceerbare manier kan beheren. Hiervoor heb ik gebruikgemaakt van GitHub en GitHub Desktop om bestanden overzichtelijk op te slaan, wijzigingen bij te houden en analyses transparant te documenteren.

Data Stewardship
Bestandsorganisatie

Om overzicht te houden heb ik een duidelijke mappenstructuur gebruikt. Hierdoor zijn alle onderdelen van het project eenvoudig terug te vinden.

Transcriptomics_RA_Final/
│
├── data/
├── scripts/
├── results/
├── figures/
└── README.md

Deze structuur zorgt ervoor dat data, scripts en resultaten gescheiden blijven en overzichtelijk georganiseerd zijn.

Versiebeheer

Voor het versiebeheer heb ik GitHub Desktop gebruikt. Tijdens het project heb ik wijzigingen opgeslagen met commits, zodat eerdere versies behouden blijven en veranderingen gevolgd kunnen worden. Hierdoor blijft duidelijk welke aanpassingen zijn uitgevoerd tijdens de analyse.

Datakwaliteit en reproduceerbaarheid

De oorspronkelijke data is niet aangepast. Analysebestanden en resultaten zijn apart opgeslagen zodat onderscheid tussen ruwe data en verwerkte data behouden blijft.

Om de analyse reproduceerbaar te maken zijn alle gebruikte scripts, invoerbestanden en resultaten opgenomen in de repository. Daarnaast beschrijft de README stap voor stap welke analyses zijn uitgevoerd.

Gebruik van GitHub

GitHub werd gebruikt als centraal platform voor het beheren van het project. Alle scripts, resultaten en figuren zijn opgeslagen in één repository, waardoor het volledige analyseproces transparant beschikbaar is.

Met GitHub Desktop konden wijzigingen eenvoudig worden bijgehouden en gesynchroniseerd. Hierdoor bleef de repository actueel en overzichtelijk.

De README bevat een beschrijving van de onderzoeksvraag, gebruikte methoden, resultaten en conclusies. Ook worden de belangrijkste bestanden en mappen toegelicht.

Reflectie

Door GitHub te gebruiken heb ik geleerd hoe belangrijk een goede bestandsstructuur en versiebeheer zijn binnen bio-informatica. Het gebruik van een centrale repository maakte het eenvoudiger om overzicht te houden en analyses reproduceerbaar te maken. Deze werkwijze sluit aan bij de principes van transparant en betrouwbaar wetenschappelijk onderzoek.
